# 📚 Documentation projet TRYBE

Ce dossier est organisé par domaine pour qu'une personne qui rejoint le projet puisse comprendre rapidement :

- d'où vient le besoin,
- quelles décisions ont été prises,
- comment l'application doit être construite.

## Structure

| Dossier | Contenu |
|---|---|
| `01-cadrage/` | Contexte, problématique, objectifs, personas |
| `02-architecture/` | Diagrammes, MVC, flux de données |
| `03-execution/` | Planning, suivi, livrables |
| `04-ux/` | Maquettes, wireframes, user flows |
| `05-sources/` | Références, liens, ressources |
